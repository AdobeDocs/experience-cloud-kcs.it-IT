## Risoluzione {#resolution}


La documentazione seguita dal cliente per integrare AEM con Smart Content è per AEM on-premise versione 6.5.

Per AEM as a Cloud Service, non è necessario configurare manualmente l’integrazione (creazione di un progetto AIO e configurazione cloud legacy). Per AEMaaCS funziona come previsto senza che venga aggiunta manualmente alcuna configurazione cloud.

Dopo aver rimosso la configurazione cloud legacy, l’apprendimento avanzato per i tag personalizzati è stato completato correttamente.

<b>RCA:</b>

La configurazione cloud legacy creata dal cliente ha interferito con l’integrazione OOTB causando l’eccezione SimilaritySearchException e causando quindi corsi di formazione archiviati.

Le configurazioni cloud legacy si trovano in /etc/cloudconfigs. In un ambiente di produzione non è possibile accedere a crx/de per rimuoverlo direttamente. Pertanto, in AEMaaCS, per un ambiente di produzione le configurazioni cloud legacy devono essere rimosse tramite una distribuzione
