# ProductAssetListener può causare SegmentNotFoundException e problemi di prestazioni

## Descrizione


`ProductAssetListener` può causare `SegmentNotFoundException` e problemi di prestazioni in AEM 6.5.6, AEM 6.5.7 e AEM 6.4.6+.



Messaggio di errore tipico di `SegmentNotFoundException` di questo problema contiene lo stacktrace come di seguito.

```
\*ERROR\* sling-oak-observation-4 org.apache.jackrabbit.oak.segment.SegmentNotFoundExceptionListener
Segment not found: 6376814a-9273-4df8-aedd-ec5e6eb7ab52. SegmentId age=238801954ms,segment-generation=GCGeneration{generation=139,fullGeneration=47,isCompacted=false}

org.apache.jackrabbit.oak.segment.SegmentNotFoundException: Segment 6376814a-9273-4df8-aedd-ec5e6eb7ab52 not found
...
at com.adobe.cq.commerce.impl.asset.ProductAssetListener.onEvent(ProductAssetListener.java:153)
```


I problemi di prestazioni possono essere lentezza, assenza di risposta e ritardo/blocco della coda di replica. Quando il problema inizia, viene visualizzato il seguente messaggio in `error.log`.

```
\*WARN\* sling-oak-observation-4 com.adobe.cq.commerce.impl.asset.ProductAssetListener NODE_MOVED event
for /content/dam/asset/metadata does not have required event info
```


I dump di thread durante il problema contengono un `sling-oak-observation` filo con `ProductAssetListner` sta correndo aggressivamente.

```
"sling-oak-observation-4" prio=5 tid=0xd7 nid=0xffffffff runnable 
...
                at com.adobe.cq.commerce.impl.asset.ProductAssetListener.onEvent(ProductAssetListener.java:153)
```

## Risoluzione


Questo problema è stato risolto con AEM 6.5.8.

[https://experienceleague.adobe.com/docs/experience-manager-65/release-notes/service-pack/previous-hotfixes-featurepacks.html?lang=en](https://experienceleague.adobe.com/docs/experience-manager-65/release-notes/service-pack/previous-hotfixes-featurepacks.html?lang=en)

L’aggiornamento dei riferimenti delle risorse di prodotto fa sì che i thread di replica siano nello stato di attesa fino al `ProductAssetListener` il thread completa i propri impegni al JCR (NPR-35269).



Per AEM 6.4.6+, disattivare il componente OSGi `com.adobe.cq.commerce.impl.asset.ProductAssetListener` è una soluzione. A questo scopo, puoi utilizzare Disabler componente in ACS AEM Commons.

[https://adobe-consulting-services.github.io/acs-aem-commons/features/osgi-disablers/component-disabler/index.html](https://adobe-consulting-services.github.io/acs-aem-commons/features/osgi-disablers/component-disabler/index.html)



Come configurare Component Disabler:

1. In CRX/DE, crea un `sling:OSGiConfig` nodo come `/apps/your project/config/com.adobe.acs.commons.util.impl.ComponentDisabler`

2. Aggiungi una proprietà String con `name = "components", value =  "com.adobe.cq.commerce.impl.asset.ProductAssetListener"`
