# Impostare Adobe I/O: Authentication - procedura dettagliata

## Descrizione

<br>Questo documento illustra come creare certificati, chiavi e token per l’esecuzione.<br><br><br><br><br>Obiettivo<br><br><br><br> <br><br>
Fornire passaggi chiari e riproducibili per generare un token Bearer di Adobe I/O al fine di eseguire chiamate API destinate a soluzioni DMA come [!DNL Target].


## Risoluzione

<br><br>Passaggi<br><br><br><br> <br><br>
<b>Passaggi per creare un token Bearer per eseguire chiamate API di Adobe I/O:</b>

1. Genera una chiave privata e un certificato pubblico
2. Crea un’integrazione all’interno della console Adobe I/O
3. Genera un token JWT
4. Scambia il token JWT con un token di accesso Bearer
5. Usa il token di accesso Bearer per eseguire chiamate API


- <b>Genera una chiave privata e un certificato pubblico seguendo la documentazione seguente:</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Su Mac, i seguenti comandi sono integrati nel terminale. 

   In un computer scarica [!DNL Cygwin] o un altro strumento a tua scelta ed eseguilo dalla riga di comando. Ecco i passaggi per scaricare e installare [!DNL Cygwin]:

   1. Passa a https://cygwin.com/install.html
   2. Scarica ed esegui `setup-x86_64.exe`

   <b>*Nota</b>: la tua home directory è `C:\cygwin64\home\WINDOWSUSER`

   Puoi cercare e installare pacchetti aggiuntivi durante la fase di installazione. È consigliabile installare tutti i componenti relativi a “`curl`” e “`ssh.`”*


   Ecco il comando da eseguire:
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Nota&lt;/b>: per generare il certificato è necessario rispondere ad alcune domande che vengono poste, come nella schermata seguente:
![rtaImage](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_0/step_par/image/rtaImage.png "rtaImage")<br> 

    Dopo aver creato i file, converti “secret.key” in “secret.pem” usando il comando seguente:
<br> <br> <br>

    | `$ openssl pkcs8 -topk8 -inform PEM -outform DER -` `in` `secret.pem  -nocrypt  secret.key` |
    | --- |

<br> 

    Di seguito sono riportati i file creati nel file system. A questo punto è possibile spostare questi file in un’altra posizione sul computer per motivi organizzativi:

- <b>Crea un’integrazione all’interno della console Adobe I/O:</b>

   - Vai a [https://console.adobe.io/](https://console.adobe.io/)
   - Seleziona un’azienda.
   - Fai clic sul pulsante “New Integration” (Nuova integrazione).

![rtaimage_2_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_1/step_par/image/rtaimage_2_.png "rtaimage_2_")
- Seleziona l’opzione “Access an API” (Accedi a un’API), quindi fai clic su “Continua”.

![rtaimage_3_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_1/step_par/image_374011898/rtaimage_3_.png "rtaimage_3_")
- Seleziona una soluzione Adobe (attualmente disponibile solo per [!DNL Target]).

    &lt;b>Nota:&lt;/b> l’utente deve essere un utente di Experience Cloud E disporre dell’accesso a tale soluzione.

- Seleziona “New Integration” (Nuova integrazione) e fai clic su “Continua”.

![rtaimage_5_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_1/step_par/image_1811594053/rtaimage_5_.png "rtaimage_5_")
- Compila il modulo di integrazione.

![rtaimage_6_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_1/step_par/image_559705621/rtaimage_6_.png "rtaimage_6_")
- Trascina “certificate.pem” dal file system al modulo per caricarlo.

![rtaimage_7_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_1/step_par/image_1707339822/rtaimage_7_.png "rtaimage_7_")
- Dopo aver caricato il file, fai clic sul pulsante “Create integration” (Crea integrazione). Vedi la schermata: fileuploaded_createint.

![rtaimage_8_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_1/step_par/image_282868645/rtaimage_8_.png "rtaimage_8_")
- Al termine dell’elaborazione, fai clic su “Continue to integration details” (Continua con i dettagli dell’integrazione).

![rtaimage_9_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_1/step_par/image_1642793077/rtaimage_9_.png "rtaimage_9_")<br>     Operazione completata. Hai creato un’integrazione.

- <b>Genera un JSON Web Token (JWT):</b>
Nell’interfaccia utente dell’integrazione, fai clic sulla scheda JWT, incolla la chiave privata e fai clic sul pulsante “Generate JWT” (Genera JWT) (vedi la schermata generazione di JWT - input).

   ![rtaimage_11_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_2/step_par/image/rtaimage_11_.png "rtaimage_11_") 
   - Al termina della creazione, viene visualizzato il token JWT e un comando CURL di esempio.
   - Fai clic sull’icona “Copy” (Copia) sotto “Generated JWT” (JWT generato).

![rtaimage_12_](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_2/step_par/image_861805817/rtaimage_12_.png "rtaimage_12_")
