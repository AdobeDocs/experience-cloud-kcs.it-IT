# Come impostare un Adobe I/O: Autenticazione - Passaggio per passaggio

## Descrizione

<br>In questo documento viene illustrato come creare certificati, chiavi e token per l&#39;esecuzione.<br><br><br><br><br>Obiettivo<br><br><br><br> <br><br>
Per fornire passaggi chiari e riproducibili per generare un token portatore Adobe I/O per eseguire chiamate API per soluzioni DMA come [!DNL Target].


## Risoluzione

<br><br>Passaggi<br><br><br><br> <br><br>
<b>Passaggi per creare un token portatore per eseguire chiamate API di Adobe I/O:</b>

1. Genera una chiave privata e un certificato pubblico
2. Creare un’integrazione all’interno della console Adobe I/O
3. Genera token JWT
4. Sostituire il token JWT con un token portatore di accesso
5. Token portatore di accesso utente per eseguire chiamate API


- <b>Genera una chiave privata e un certificato pubblico seguendo la seguente documentazione:</b> https://www.adobe.io/apis/cloudplatform/console/authentication/createcert.html


   Su un Mac, i seguenti comandi sono integrati nel terminale. 

   Su un computer, è necessario scaricare [!DNL Cygwin] (o altro strumento di preferenza personale) ed eseguirlo dalla riga di comando. Segui i passaggi per scaricare e installare [!DNL Cygwin]:

   1. Sfoglia https://cygwin.com/install.html
   2. Scarica ed esegui `setup-x86_64.exe`

   <b>*Nota</b>:  La tua home directory è: `C:\cygwin64\home\WINDOWSUSER`

   È possibile cercare e installare pacchetti aggiuntivi durante la fase di installazione. Consiglio di installare tutto ciò che riguarda &quot;`curl`&quot; e &quot;`ssh.`&quot;*


   Ecco il comando da eseguire:
   <br> <br> <br>

   | `$ openssl req -nodes -text -x509 -newkey rsa:2048 -keyout secret.pem -out certificate.pem -days 356` |
   | --- |

<br> 

    &lt;b>Nota&lt;/b>:  Ti fa diverse domande che devi compilare per generare il certificato, vedi la schermata seguente:
![rtaImage](https://helpx.adobe.com/content/dam/help/en/marketing-cloud-core/kb/adobe-io-authentication-step-by-step/jcr%3acontent/main-pars/procedure/proc_par/step_0/step_par/image/rtaImage.png "rtaImage")<br> 

    Dopo aver creato i file, devi convertire `secret.key` in `secret.pem` utilizzando il seguente comando:
<br> <br> <br>

    | `$ openssl pkcs8 -topk8 -informa PEM -outform DER -` `in` `secret.pem -nocrypt secret.key` |
    | — |
